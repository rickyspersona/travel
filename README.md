# Travel Reports

兩個 sub-folder 各自存放 HTML 旅遊 / 出差報告。

```
travel/
├── index.html             ← root landing page (Vercel 預設入口)
├── family-trip/           ← 家庭旅遊
│   └── demo.html
└── business-trip/         ← 商務出差
    └── demo.html
```

## Vercel 部署

1. 進 https://vercel.com 並登入(GitHub OAuth 最快)
2. **Add New → Project** → Import `rickyspersona/travel`
3. 不用改任何設定,直接 Deploy

部署後 URL 結構(假設 subdomain 是 `travel-xxx.vercel.app`):

| 路徑 | 用途 |
|---|---|
| `/` | root landing page,列出兩個子資料夾入口 |
| `/family-trip/demo.html` | family-trip 範例 |
| `/business-trip/demo.html` | business-trip 範例 |

## 之後新增報告

直接 commit/push 新 HTML, Vercel 會自動重新部署:

```bash
cd ~/Documents/travel
# 新增 family-trip 報告
cp ~/path/to/new-report.html family-trip/kyoto-2026.html
git add . && git commit -m "Add kyoto trip report" && git push
```

## Vercel 行為備忘

- 純 static hosting,所有檔案原樣 serve
- subfolder 名稱不能跟 Vercel 保留字衝突(`api/`、`_next/` 等),`family-trip/` 跟 `business-trip/` 安全
- 不像 GitHub Pages 對根目錄有特殊處理
